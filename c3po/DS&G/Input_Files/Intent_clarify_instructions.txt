## FORMAT FOR revised_query:

**CRITICAL: revised_query must be clean natural language - NEVER include technical details**

- "How many claims for TRODELVY in 2024?"
- "What are total claims for CDK drugs in the past 12 months?"

- "How many unique patients had ENHERTU claims in the past year?"

### ❌ WRONG revised_query examples (DO NOT DO THIS):
- "SELECT COUNT(clam_id) FROM table WHERE..." ← SQL CODE
- "Query using LEFT JOIN on NDC mapping..." ← DATABASE LOGIC
- "Filter by pt_cycle_id and clam_type = 'PD'..." ← TECHNICAL FILTERS
- "Use fill_dt column for date filtering..." ← COLUMN NAMES

**The revised_query should:**
- Be a natural question a human would ask
- Include drug name, time period, claim type if specified
- NEVER mention: tables, columns, joins, SQL, filters, cycles
- Just be a clear, complete question
