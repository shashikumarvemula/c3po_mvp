    STEP 3: APPLY STATISTICAL OUTLIER DETECTION
    --------------------------------------------
        USE IQR METHOD FOR ALL OUTLIER DETECTION QUERIES:
        
        METHOD: IQR (INTERQUARTILE RANGE) ANALYSIS (Universal Method for All Outlier Detection)
            WHEN USER ASKS about any type of outlier detection - trends, comparisons, ratios, or changes.

            Formula: 
            - Q1 = 25th percentile
            - Q3 = 75th percentile  
            - IQR = Q3 - Q1
            - Lower Bound = Q1 - 1.5 × IQR
            - Upper Bound = Q3 + 1.5 × IQR

            SIMPLIFIED OUTLIER CLASSIFICATION:
            Use ONLY three categories: 'Low Outlier', 'Normal', 'High Outlier'

            SCENARIO A: TIME-SERIES OUTLIER DETECTION (Monthly/Quarterly Claims)
            Apply IQR directly on claim counts or metric values.
            REMEMBER: Data already has 2-month lag applied from STEP 2.

            SQL Pattern:
            ```sql
            WITH data_aggregated AS (
                -- Your standard aggregation query with all filters AND 2-MONTH LAG
                SELECT dimension, metric_value FROM ...
            ),
            quartiles AS (
                SELECT 
                    PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY metric_value) as q1,
                    PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY metric_value) as median,
                    PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY metric_value) as q3
                FROM data_aggregated
            ),
            bounds AS (
                SELECT 
                    q1,
                    q3,
                    median,
                    (q3 - q1) as iqr,
                    q1 - 1.5 * (q3 - q1) as lower_bound,
                    q3 + 1.5 * (q3 - q1) as upper_bound
                FROM quartiles
            )
            SELECT 
                d.*,
                b.median,
                b.q1,
                b.q3,
                b.iqr,
                b.lower_bound,
                b.upper_bound,
                CASE 
                    WHEN d.metric_value > b.upper_bound THEN 'High Outlier'
                    WHEN d.metric_value < b.lower_bound THEN 'Low Outlier'
                    ELSE 'Normal'
                END as outlier_flag,
                ((d.metric_value - b.median) / NULLIF(b.median, 0)) * 100 as pct_deviation_from_median
            FROM data_aggregated d
            CROSS JOIN bounds b
            ORDER BY 
                CASE 
                    WHEN d.metric_value > b.upper_bound THEN d.metric_value - b.upper_bound
                    WHEN d.metric_value < b.lower_bound THEN b.lower_bound - d.metric_value
                    ELSE 0
                END DESC
            ```

            SCENARIO B: BATCH COMPARISON OUTLIER DETECTION
            Apply IQR on percentage changes between batches.
            REMEMBER: Data already has 2-month lag applied from STEP 2.

            SQL Pattern:
            ```sql
            WITH batch_comparison AS (
                SELECT 
                    dimension,
                    batch1_value,
                    batch2_value,
                    batch1_value - batch2_value as absolute_change,
                    CASE WHEN batch2_value > 0 
                        THEN ((batch1_value - batch2_value)::FLOAT / batch2_value) * 100
                        ELSE NULL 
                    END as pct_change
                FROM ...
            ),
            pct_change_quartiles AS (
                SELECT 
                    PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY pct_change) as q1,
                    PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY pct_change) as median,
                    PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY pct_change) as q3
                FROM batch_comparison
                WHERE pct_change IS NOT NULL
            ),
            pct_change_bounds AS (
                SELECT 
                    q1,
                    q3,
                    median,
                    (q3 - q1) as iqr,
                    q1 - 1.5 * (q3 - q1) as lower_bound,
                    q3 + 1.5 * (q3 - q1) as upper_bound
                FROM pct_change_quartiles
            )
            SELECT 
                bc.*,
                b.median as median_pct_change,
                b.q1,
                b.q3,
                b.iqr,
                b.lower_bound,
                b.upper_bound,
                CASE 
                    WHEN bc.pct_change > b.upper_bound THEN 'High Outlier'
                    WHEN bc.pct_change < b.lower_bound THEN 'Low Outlier'
                    ELSE 'Normal'
                END as outlier_flag,
                ((bc.pct_change - b.median) / NULLIF(b.median, 0)) * 100 as deviation_from_median_pct
            FROM batch_comparison bc
            CROSS JOIN pct_change_bounds b
            WHERE bc.pct_change IS NOT NULL
            ORDER BY 
                CASE 
                    WHEN bc.pct_change > b.upper_bound THEN bc.pct_change - b.upper_bound
                    WHEN bc.pct_change < b.lower_bound THEN b.lower_bound - bc.pct_change
                    ELSE 0
                END DESC
            ```

            SCENARIO C: RATIO OUTLIER DETECTION (RX/MX Ratios)
            Apply IQR on ratio values.
            REMEMBER: Data already has 2-month lag applied from STEP 2.

            SQL Pattern:
            ```sql
            WITH ratio_calc AS (
                SELECT 
                    period,
                    rx_claims,
                    mx_claims,
                    CASE WHEN mx_claims > 0 
                        THEN rx_claims::FLOAT / mx_claims 
                        ELSE NULL

                SELECT 
                    PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY rx_mx_ratio) as q1,
                    PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY rx_mx_ratio) as median,
                    PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY rx_mx_ratio) as q3
                FROM ratio_calc
                WHERE rx_mx_ratio IS NOT NULL
            ),
            ratio_bounds AS (
                SELECT 
                    q1,
                    q3,
                    median,
                    (q3 - q1) as iqr,
                    q1 - 1.5 * (q3 - q1) as lower_bound,
                    q3 + 1.5 * (q3 - q1) as upper_bound
                FROM ratio_quartiles
            )
            SELECT 
                r.*,
                b.median,
                b.q1,
                b.q3,

            CROSS JOIN ratio_bounds b
            WHERE r.rx_mx_ratio IS NOT NULL
            ORDER BY 
                CASE 
                    WHEN r.rx_mx_ratio > b.upper_bound THEN r.rx_mx_ratio - b.upper_bound
                    WHEN r.rx_mx_ratio < b.lower_bound THEN b.lower_bound - r.rx_mx_ratio
                    ELSE 0
                END DESC
            ```
